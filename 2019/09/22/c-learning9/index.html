指针

运算符 &
获得变量的地址，它的操作数必须是变量

&不能对没有地址的东⻄取地址

• &(a+b)
• &(a++)
• &(++a)

指针

就是保存地址的变量

int* p = &i;

星号表示p是一个指针 它指向一个int 把i地址交给p

  • int* p,q;
  • int *p,q;
  • 都是在说p是指针变量 而q是普通变量

指针变量

变量的值是内存的地址

• 普通变量的值是实际的值
• 指针变量的值是具有实际值的变量的地址

作为参数的指针

  • 格式：void f(int *p);
  • 在被调⽤的时候得到了某个变量的地址：
    • int i=0; f(&i); 交给他地址 用&符号
      • 在函数⾥⾯可以通过这个指针访问外⾯的这个i

访问那个地址上的变量*

如何从函数传出值：
*是⼀个单目运算符，⽤来访问指针的值所表⽰的地址上的变量

*p要看做一个整体 看成一个整数
a[0]也是一个整体

＊左值之所以叫左值

• 是因为出现在赋值号左边的不是变量，⽽是值，是表达式计算的结果：

• a[0] = 2;
• *p = 3; 取得p这个指针地址所代表的变量 是表达式运算的结果 放在赋值号左边来接受结果

• 是特殊的值，所以叫做左值

指针的运算符&*

  • 互相反作⽤

    *&yptr -> * (&yptr) -> * (yptr的地
    址)-> 得到那个地址上的变量 -> yptr

& * yptr -> &(*yptr) -> &(y) -> 得到y的地
址，也就是yptr -> yptr

指针调用

  1. 利用函数交换两个数的值

2.当需要函数返回多个值，某些值就只能通过指针返回

  • 传⼊的参数实际上是需要保存带回的结果的变量

*p不能直接赋值

int *p
*p = 数 不可以这么写

任何地址变量没有被赋值之前 没有得到任何实际变量地址之前 不能通过用星号去访问任何数据

传入函数的数组

函数参数表中的数组实际上是指针

  • sizeof(a) == sizeof(int*)
  • 但是可以⽤用数组的运算符[]进⾏行运算 在运算的时候可以写a[]=啥啥啥

数组参数：

int sum(int *ar, int n);=int sum(int *, int);=int sum(int ar[], int n); =int sum(int [], int);

*ar = ar[]

数组变量是特殊的指针

数组变量本⾝身表达地址，所以
- int a[10]; int*p=a; // 无需⽤&取地址
- 但是数组的单元表达的是变量，需要⽤用&取地址

[]运算符可以对数组做，也可以对指针做:
- p[0] <==> a[0] = *p

数组的有效下标等于其定义的数-1； 因为从0开始计数

*运算符可以对指针做，也可以对数组做:

  • *a = a[0];

指针与const

指针是一种变量 有指针本身和指针所指的变量

指针本身是const 所指变量也可以是const

指针是const 表⽰一旦得到了某个变量的地址，不能再指向
其他变量

  • int * const q = &i;// q是const i的地址=q的值 q不能再指向别人 q只指向i
  • *q = 26;// OK q做访问没问题 因为q所指的i不是const
  • q++; // ERROR 因为q是const所以q++不能做

判断哪个被const了的标志是const在*的前⾯面还是后面

const 数组：

const int a[] = {1,2,3,4,5,6,};
数组变量已经是const的指针了，这⾥里的const 表明数组的每个单元都是const int

所以必须通过初始化进⾏行赋值

保护数组值

因为把数组传⼊入函数时传递的是地址，所以那个函数内部可以修改数组的值

为了保护数组不被函数破坏，可以设置参数为const int sum(const int a[], int length);

数组运算

指针是可计算的

给⼀个指针加1表⽰要让指针指向下⼀个变量

指针计算

这些算术运算可以对指针做:

  • 给指针加、减⼀一个整数(+, +=, -, -=)
    • 减往前挪 加往后挪
  • 递增递减(++/—) p++ p–
  • 两个指针相减
    • 两个地址的差除以sizeof的类型 这两个地址中间能放几个这样类型的东西

*p++

++运算符比星高
*p++是 p+1以前的结果 之后再对他取星 即：取出这个P指针所指的地方的东西 取完之后把p移到下一个位置去

指针比较

  • <, <=, ==, >, >=, != 都可以对指针做
  • 比较两个指针在内存中的地址的大小
  • 数组中的单元的地址肯定是线性递增的

0地址

当然你的内存中有0地址，但是0地址通常是个不能随便碰的地址 所以你的指针不应该具有0值

因此可以⽤用0地址来表示特殊的事情:

  • 返回的指针是无效的
  • 指针没有被真正初始化(先初始化为0)

有的编译器不愿意你⽤0来表示0地址
•
NULL是一个预定定义的符号，表示0地址

##指针的类型

⽆论指向什么类型，所有的指针的⼤小都是⼀样的，因为都是地址
但是指向 不同类型的指针是不能直接互相赋值的
这是为了避免⽤用错指针

void*

我有一个指针 他指向一块内存 但是不清楚他指向的类型是什么

指针作用

  • 需要传⼊较大的数据时用作参数 比如传数组
  • 传⼊数组后对数组做操作
    • 函数返回不⽌一个结果 调用函数时 用指针带出结果
      •需要⽤函数来修改不⽌一个变量
  • 动态申请的内存

动态内存分配

输入数据：

int a = (int)malloc(n*sizeof(int));

调用malloc的时候要包括<Stdlib.h>

向malloc申请的空间的⼤大⼩小是以字节为单位的

返回的结果是void*，需要类型转换为⾃自⼰己 需要的类型

- (int*)malloc(n*sizeof(int))

好习惯： 定义一个指针的同时把他初始为零

int *p=0;

free(p);

没空间：

如果申请失败则返回0，或者叫做NULL

常⻅问题

  • 申请了没free—>⻓时间运行内存逐渐下降
    新手:忘了
    ⽼手:找不到合适的free的时机
  • free过了再free
  • 地址变过了，直接去free

写一个malloc就free掉
