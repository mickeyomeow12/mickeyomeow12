函数定义

函数是⼀块代码，接收零个或多个参数，
做⼀件事情，并返回零个或⼀个值

void 返回类型 （这个函数没有返回结果） sum函数名 （ 参数表 列出一个个参数 逗号分隔 类型+变量名字 ）

{
函数体
}

调用函数

函数名（参数值 函数参数表里面有几个参数 就需要传递几个参数 而这个参数值是要传递进函数的参数表里的 一一对应）；

函数名(参数值);

  • ( )起到了表⽰函数调⽤的重要作⽤
    • 即使没有参数在调用的时候也需要写出()符号
  • 如果有参数，则需要给出正确的数量和顺序
  • 这些值会被按照顺序依次⽤来初始化函数中的参数

函数使用

如果函数需要返回一个结果 那么需要用return 把结果交给调用他的地方

return的作用

return停⽌函数的执⾏，并送回⼀个值

  • return;
  • return 表达式;（可以带一个值 把值交给调用这个函数的地方 这个值不只是1 0 而是看函数里写的是什么 return对应的变量里是什么 在函数里的值是什么则传回什么值）

从函数中返回值

可以赋值给变量
- 可以再传递给函数
- 甚⾄可以丢弃
- 有的时候要的是副作⽤

没有返回值的函数

  • void 函数名(参数表)
  • 不能使⽤带值的return
    • 可以没有return
  • 调⽤的时候不能做返回值的赋值

函数先后关系

最好是先写函数再写主函数

这样把sum()写在上⾯，是因为：

  • C的编译器⾃上⽽下顺序分析你的代码
  • 在看到sum(1,10)的时候，它需要知道sum()的样⼦
    只有在前头看到了sum是什么样的才可以在主函数的调用的时候已经知道sum函数的样子
    • 也就是sum()要⼏个参数，每个参数的类型如何，返回什么类型
  • 这样它才能检查你对sum()的调⽤是否正确

如果先写主函数的话需要先写自定义的函数声明

函数原型

  • 函数头，以分号“;”结尾，就构成了函数的原型

  • 函数原型的目的是告诉编译器这个函数⻓什么样

    • 名称
    • 参数（数量及类型）
    • 返回类型

  • 函数头在下面

  • 函数头有分号就是函数原型

  • 旧标准习惯把函数原型写在调⽤它的函数⾥⾯

  • 现在⼀般写在调⽤它的函数前⾯

  • 原型声明⾥可以不写参数（自定函数中变量且是在主函数调用进的变量）的名字，但是⼀般仍然写上 方便阅读

函数传递

调用函数

如果函数有参数，调⽤函数时必须传递给它数量、类型正确的值

可以传递给函数的值是表达式的结果，这包括：
• 字⾯量
• 变量
• 函数的返回值
• 计算的结果

本地变量

函数的每次运⾏行，就产⽣生了⼀一个独⽴立的变量空间，在这个空间中的变量，
是函数的这次运⾏行所独有的，称作本地变量

定义在函数内部的变量就是本地变量

参数也是本地变量

变量的⽣存期和作用域

生存期:什么时候这个变量开始出现了，到什么时候
它消亡了

作用域:在(代码的)什么范围内可以访问这个变量
(这个变量可以起作用)

对于本地变量，这两个问题的答案是统一的:⼤括号

——块

如果变量不存在：not found in current context

看变量什么时候消亡什么时候存在

在swap里头main里头的a和b就不存在了 生存还在但main的ab不在这里作用

进了swap的大括号里 txy即tab（swap）里的存在且作用 出了就没用了

本地变量的规则

本地变量时定义在块内的
- 定义在函数内
- 定义在语句内
- 随意拉一对大括号来定义变量

*程序运⾏进入这个块之前，其中的变量不存在，离开这个块，其中的变量
就消失了 *

块中的变量仅在块里作用

块外⾯面定义的变量在⾥里⾯面仍然有效

块⾥里⾯面定义了和外⾯面同名的变量则掩盖了外⾯面的 在块里的时候直接覆盖了 出了块之后又恢复了

不能在⼀一个块内定义同名的变量

本地变量不会被默认初始化

参数在进⼊入函数的时候被初始化了

不要写void 函数名(空); 函数名后面的（）一定要写全参数类型；

c语言不允许函数里嵌套函数 不能放body

return 0;是有意义的
